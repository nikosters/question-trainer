# Формат JSON-пакета

Пакет заданий - это JSON-файл с корневым объектом и массивом `questions`.

```json
{
  "questions": []
}
```

Поддерживаются два типа заданий:

- `multiple_choice` - вопрос с четырьмя вариантами ответа
- `formula_assembly` - восстановление формулы по фрагментам

Если поле `type` отсутствует, приложение считает задание обычным `multiple_choice`.

## Вопрос с вариантами ответа

```json
{
  "questions": [
    {
      "id": "q1",
      "type": "multiple_choice",
      "question": "Решите уравнение: $2x + 5 = 17$",
      "option_a": "$x = 5$",
      "option_b": "$x = 6$",
      "option_c": "$x = 7$",
      "option_d": "$x = 8$",
      "correct_option": "B",
      "explanation": "Переносим 5: $2x = 12$, значит $x = 6$.",
      "topic": "Линейные уравнения",
      "difficulty": "Базовый"
    }
  ]
}
```

Обязательные поля:

- `id` - непустая строка, уникальная внутри пакета
- `question` - текст вопроса
- `option_a` - вариант A
- `option_b` - вариант B
- `option_c` - вариант C
- `option_d` - вариант D
- `correct_option` - одна из строк `A`, `B`, `C`, `D`
- `explanation` - объяснение ответа

## Восстановление формулы

```json
{
  "questions": [
    {
      "id": "formula_triangle_area",
      "type": "formula_assembly",
      "question": "Восстановите формулу площади треугольника через основание и высоту.",
      "formula_parts": [
        { "latex": "S" },
        { "latex": "=" },
        { "blank": "coef" },
        { "latex": "a" },
        { "latex": "h" }
      ],
      "blanks": [
        { "id": "coef", "answer": "\\frac{1}{2}" }
      ],
      "distractors": [
        "2",
        "\\frac{1}{3}",
        "\\frac{1}{4}"
      ],
      "explanation": "$S = \\frac{1}{2}ah$.",
      "topic": "Планиметрия",
      "difficulty": "Базовый"
    }
  ]
}
```

Обязательные поля:

- `id` - непустая строка, уникальная внутри пакета
- `type` - строка `formula_assembly`
- `question` - текст вопроса
- `formula_parts` - непустой массив фиксированных фрагментов и пропусков
- `blanks` - непустой массив правильных ответов для пропусков
- `explanation` - объяснение ответа

Элементы `formula_parts`:

- `{ "latex": "..." }` - фиксированный LaTeX-фрагмент
- `{ "blank": "blank_id" }` - интерактивный пропуск

Элементы `blanks`:

- `id` - непустой идентификатор пропуска
- `answer` - непустой raw LaTeX-фрагмент без `$...$`

Необязательное поле `distractors` содержит дополнительные неправильные raw LaTeX-фрагменты для банка ответов.

## Необязательные поля

- `topic` - тема вопроса
- `difficulty` - сложность вопроса

Если `topic` или `difficulty` отсутствуют, приложение считает их пустыми строками.

## Формулы

В `question`, вариантах ответа и `explanation` inline-формулы пишутся в формате `$...$`.

В `formula_parts`, `blanks.answer` и `distractors` используется raw LaTeX без `$...$`.

Пример:

```json
"question": "Найдите производную функции $f(x) = x^3 - 4x$"
```

## Шаблон для генерации

```json
{
  "questions": [
    {
      "id": "formula_1",
      "type": "formula_assembly",
      "question": "Восстановите формулу ...",
      "formula_parts": [
        { "latex": "" },
        { "blank": "part_1" },
        { "latex": "" }
      ],
      "blanks": [
        { "id": "part_1", "answer": "" }
      ],
      "distractors": [],
      "explanation": "",
      "topic": "",
      "difficulty": ""
    }
  ]
}
```

Перед импортом заполните все пустые строки: приложение отклоняет пустые обязательные поля.

## Ошибки валидации

Импорт будет отклонен, если:

- корень JSON не является объектом
- `questions` отсутствует или не является массивом
- массив `questions` пустой
- вопрос не является объектом
- `type` неизвестен
- обязательное поле отсутствует или является пустой строкой
- `correct_option` не равен `A`, `B`, `C` или `D`
- `formula_parts` отсутствует, пустой или содержит элементы не в формате `{ "latex": "..." }` / `{ "blank": "..." }`
- `blanks` отсутствует, пустой или содержит повторяющиеся `id`
- пропуск из `formula_parts` не описан в `blanks`
- пропуск из `blanks` не используется в `formula_parts`
- `distractors` не является массивом непустых строк
- внутри пакета повторяется `id`
